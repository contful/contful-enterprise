## 报告漏洞

如果你发现了安全漏洞，**请不要在公开 Issue 中报告**。请通过以下方式私下报告：

- 发送邮件至 **[hi@reepu.com](mailto:hi@reepu.com)**

请在你的报告中包含以下信息：

- 漏洞的详细描述
- 复现步骤
- 受影响版本
- 可能的修复建议（如有）

### 响应时间承诺

- **48 小时内**：确认收到报告
- **7 天内**：提供修复方案或补丁
- **30 天内**：公开发布安全公告（如适用）

## 安全特性

Contful 内置以下安全机制：

| 特性                 | 说明                                      |
| -------------------- | ----------------------------------------- |
| 非对称加密           | RSA / SM2 登录加密，保护传输安全          |
| 双因子认证（MFA）    | TOTP 标准，增强账户安全                   |
| HMAC 防篡改          | 审计日志完整性校验，防止日志被篡改        |
| RBAC 权限体系        | 细粒度角色权限控制，最小权限原则          |
| 配置加密存储         | 敏感配置通过 AES-256-GCM 加密存储         |
| JWT 认证             | 无状态认证，支持 Token 刷新与吊销         |

## 安全最佳实践

部署 Contful 时，建议遵循以下最佳实践：

1. **生产密钥**：设置 32 字节随机 `SECRET` 和 `CONTFUL_CONFIG_MASTER_KEY`
2. **HTTPS**：始终通过 HTTPS 提供服务
3. **数据库**：使用强密码，限制网络访问
4. **定期更新**：保持依赖和运行环境为最新版本
5. **日志审计**：定期检查审计日志，关注异常操作
